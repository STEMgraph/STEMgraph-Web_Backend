Projekt: STEMgraph · Lizenz: MIT · Kontakt: GitHub

Überblick

Die STEMgraph API stellt einen REST-basierten Zugriff auf eine graphorientierte Datenbank von STEM-Übungen bereit. Die Daten liegen intern im JSON-LD-Format vor und können in verschiedenen Ausgabeformaten abgerufen werden. Die Schnittstelle unterstützt Filterung nach Autoren, Keywords und Themen, das Traversieren von Abhängigkeiten im Lernpfad sowie die Aggregation von Metadaten wie Autoren- und Keyword-Statistiken.

Die API ist über einen Entwicklungsserver unter http://localhost:8000 und über den Produktivserver unter https://stemgraph-api.boekelmann.net erreichbar.

Wichtige Endpunkte

GET /
API-Root
Gibt eine einfache Begrüßungsnachricht zurück und dient als schneller Health-Check, ob der Dienst läuft.
Antwort: 200 OK, JSON-Objekt mit Feld message.

Übungen (exercises)

GET /exercises
Übungen mit optionalen Filtern auflisten
Liefert eine Liste von Übungen aus der Datenbank. Die Ergebnisse können nach Autor, Keyword oder Thema gefiltert werden und in verschiedenen Formaten (JSON-LD, Node-Link, YAML) zurückgegeben werden. Die Filter unterstützen exakte und partielle Übereinstimmung.
Parameter Typ Beschreibung Beispiel
author string (optional) Filtert nach Autorname. John Doe
keyword string (optional) Filtert nach Keyword. Python
topic string (optional) Filtert nach Thema aus dem Feld teaches. Functions
match string (optional) Matching-Modus: exact oder partial (Standard: exact). partial
format string (optional) Ausgabeformat: jsonld, nodelink oder yaml (Standard: jsonld). nodelink
Antworten: 200 OK (Graph in gewünschtem Format), 400 Bad Request bei ungültigen Parametern, 500 Internal Server Error bei Serverfehlern.
GET /exercises/start-nodes
Startknoten ohne Abhängigkeiten
Liefert alle Übungen, die keine eingehenden Abhängigkeiten besitzen und damit als Einstiegspunkte in den Lernpfad dienen. Das Ausgabeformat kann wie bei /exercises gewählt werden.
GET /exercises/end-nodes
Endknoten ohne ausgehende Abhängigkeiten
Liefert alle Übungen, die keine ausgehenden Abhängigkeiten besitzen und damit Endpunkte im Lernpfad darstellen.
GET /exercises/{uuid}
Einzelne Übung abrufen
Ruft eine spezifische Übung anhand ihrer UUID ab und gibt sie als kleinen Graphen mit einem einzelnen Knoten zurück. Das Format kann wie gewohnt gewählt werden.
Pfadparameter Typ Beschreibung
uuid string (UUID) Eindeutige Kennung der Übung.
Antworten: 200 OK (Übung gefunden), 404 Not Found (nicht vorhanden), 500 Internal Server Error.
GET /exercises/{uuid}/path
Abhängigkeitskette zu einer Übung
Gibt den vollständigen Abhängigkeitsweg (Prerequisite-Kette) zurück, der zu einer bestimmten Übung führt. So lässt sich nachvollziehen, welche Übungen zuvor abgeschlossen werden sollten.

Graph (graph)

GET /graph
Kompletten Graph abrufen
Liefert den gesamten Übungsgraphen mit allen Knoten und Kanten. Für große Datenbestände kann die Antwort entsprechend umfangreich sein. Das Ausgabeformat ist wählbar.
GET /graph/statistics
Graphstatistiken abrufen
Liefert aggregierte Statistiken über den Graphen, zum Beispiel die Gesamtzahl der Übungen, die Anzahl unterschiedlicher Keywords und die Gesamtzahl der Keyword-Vorkommen.

Autoren (authors)

GET /authors
Alle Autoren auflisten
Gibt eine Liste aller eindeutigen Autoren zurück, die in der Datenbank vorkommen, inklusive Metadaten im JSON-LD-Stil.
GET /authors/count
Autoren-Häufigkeiten
Liefert eine Häufigkeitsverteilung, wie viele Übungen pro Autor vorhanden sind.

Keywords (keywords)

GET /keywords
Alle Keywords auflisten
Gibt eine Liste aller eindeutigen Keywords zurück, die in der Datenbank verwendet werden.
GET /keywords/count
Keyword-Häufigkeiten
Liefert eine Häufigkeitsverteilung, wie oft jedes Keyword in den Übungen vorkommt.

Administration (admin)

POST /admin/refresh-db
Datenbank-Refresh aus GitHub anstoßen
Startet eine Hintergrundaufgabe, die die Übungsdatenbank aus den GitHub-Repositories aktualisiert. Dabei werden Metadaten aus den Repositories gelesen, gecacht und die JSON-LD-Datenbank neu erzeugt. In der aktuellen Version ist der Endpunkt nicht authentifiziert; für einen produktiven Einsatz sollte eine Authentifizierung (z. B. API-Key) ergänzt werden.
Antwort: 202 Accepted mit Statusmeldung, oder 500 Internal Server Error bei Fehlern.

Datenmodelle (Schemas)

Exercise Knoten

Ein Exercise repräsentiert eine einzelne Übung im STEMgraph-System. Jede Übung besitzt eine eindeutige ID, einen Typ, einen Namen und optionale Metadaten wie Beschreibung, Keywords, Autoren und Lernziele.

  • @id: eindeutige Kennung (URI/UUID)
  • @type: immer Exercise
  • name: Titel der Übung
  • description: optionale Beschreibung
  • keywords: Liste von Schlagwörtern
  • author: Liste von Objekten mit name und optional url
  • teaches: Lernziele/Themen (String oder Liste)
  • requires: IDs von Voraussetzungen (optional)

JsonLdGraph Primärformat

Das JsonLdGraph-Schema beschreibt den internen JSON-LD-Graphen. Er besteht aus einem @context und einem @graph, das eine Liste von Exercise-Knoten enthält.

NodeLinkGraph Visualisierung

Das NodeLinkGraph-Schema beschreibt eine Node-Link-Darstellung, wie sie z. B. von D3.js verwendet wird. Es enthält eine Liste von nodes und eine Liste von links, die Kanten zwischen den Knoten repräsentieren.

Statistics

Das Statistics-Schema fasst Kennzahlen über den Graphen zusammen, darunter die Anzahl der Knoten, die Anzahl unterschiedlicher Keywords und die Gesamtzahl der Keyword-Vorkommen sowie Zeitstempel zur Erstellung und letzten Änderung.

Error

Das Error-Schema beschreibt standardisierte Fehlermeldungen mit einem Textfeld detail und einem HTTP-Statuscode status. Es wird in den vordefinierten Antworten NotFound, BadRequest und InternalServerError verwendet.
